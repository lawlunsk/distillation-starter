# Classification

Use this guide to decide what kind of note, if any, a highlight should become.

## CONCEPT

A concept explains a reusable mechanism or distinction. It should be understandable without rereading the source.

Signals:

- explains why something happens
- applies across contexts
- has limits or counterexamples
- can support multiple production angles
- is not just a thesis or slogan

## PIVOT

A pivot maps a territory. It clarifies tensions, distinctions, and open questions across several concepts.

Signals:

- the source opens a whole field
- several notes orbit the same domain without a meta-note
- the right title is a domain or question, not a claim

Create pivots sparingly and with confirmation.

## FRAMEWORK

A framework is a reusable thinking tool.

Signals:

- dimensions, matrix, criteria, or question set
- useful before a decision
- execution depends on context

## HOWTO

A how-to is a repeatable procedure.

Signals:

- fixed sequence of actions
- each step is executable
- common mistakes are identifiable

## MAP

A map is navigation, not argument. It groups related notes in a domain.

Create maps only with confirmation.

## WATCHLIST

Use a watchlist or inbox note for tools, releases, benchmarks, market signals, or references that may matter later but are not durable knowledge yet.

## NOISE

Reject material that is:

- anecdote without mechanism
- claim without support
- isolated fact
- duplicate with no new angle

Ask before discarding if the user's intent is unclear.
