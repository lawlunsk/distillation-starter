# Designing useful friction

Friction is not always a defect. In tools that shape attention, a small pause can prevent accidental escalation.

The best systems remove mechanical friction while preserving decision friction. A confirmation step before deleting a database is useful; a confirmation step before every text edit is not.

Design teams often confuse speed with flow. Flow means the user can continue their intention without avoidable interruption, not that every action is instant.
